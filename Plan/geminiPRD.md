# PRD: Open-Source Local-First Research Hub (Zero-API Edition)

## 1. Product Philosophy
**"Zero SaaS. Zero API Fees. 100% Local AI."**
This tool provides a professional UX Research analysis environment by pairing a local Next.js interface ("Report Builder") with the user's existing AI IDE (Cursor/Claude Code). The Next.js app handles the complex UX (video playback, transcript highlighting, drag-and-drop), while the IDE handles the heavy AI computation (thematic analysis, tagging, formatting).

## 2. Tech Stack (The Google/Notion Pivot)
*   **Frontend/Local Server:** Next.js (App Router), Tailwind CSS, React Split Pane.
*   **AI Engine:** Cursor IDE or Claude Code (Local context).
*   **Storage (Raw Video):** Google Drive (or Local `/public` folder).
*   **Publishing Target:** Notion (via Notion API) or GitHub Pages.
*   **Video Slicer (For HTML Export):** Local `ffmpeg` (spawned via Next.js local API route).
*   **Database:** Local JSON and Markdown files.

## 3. Core Workflows & Features

### Phase 1: Ingestion & The First Draft
1.  **Setup:** User drops Zoom `.mp4` and `.vtt`/`.txt` transcripts into the project folder.
2.  **Context Building:** Cursor prompts the user for the Research Plan, Background, and global/custom **Codebook**. Saves to `project.json`.
3.  **Initial AI Pass:** Cursor reads the transcripts and writes the first draft of `findings.md`. It extracts initial quotes and formats them as markdown clips.

### Phase 2: The "Report Builder" UI (The Core Workspace)
The user opens `localhost:3000/builder`. The UI is split into two main panes:

**Left Pane (The Source):**
*   **Video Player:** Sticky at the top.
*   **Transcript Viewer:** Scrollable text below the video. Each row has a timestamp. Clicking a row plays the video from that time.
*   **Interactive Highlighting:** User highlights text -> clicks "+ Clip" modal -> Text visually transforms into a "Quote Card" on the transcript.

**Right Pane (The Output):**
*   **View Toggle:** Switch between `Formatted View` (Rendered MD) and `Raw View` (Text editor).
*   **Drag & Drop:** User drags the newly created Quote Card from the left pane into the `Raw View` on the right to insert the clip into the markdown.
*   **Quote Cards (Visual):** Display main transcript text, timestamp, session ID, and **Tags**. 
*   **Interaction:** Clicking a quote card timestamp plays the video and *auto-stops* at the end of the duration.

### Phase 3: The "Magic AI Loop" (Zero-API Syncing)
This is the core innovation. The UI and the IDE work in tandem via the local file system.

1.  **AI Modification:** The user wants to dig deeper into a theme. They go to Cursor and type: *"Review the transcript again and add clips related to 'Onboarding Friction' to findings.md."*
2.  **The Sync Button:** Cursor updates the file. The user clicks a custom **"Refresh UI"** button (not F5) in the Report Builder.
3.  **UI Updates:** The Right Pane re-renders the new Markdown. The Left Pane (Transcript) automatically highlights the new quote cards generated by the AI.

### Phase 4: Tag Board Flow
1.  **View Toggle:** User switches from "Findings" to "Tags" in the global header.
2.  **Prompt Generator:** A modal pops up providing the exact prompt to copy-paste into Cursor:
    *   *Prompt A:* "Parse `findings.md` and reorganize all clips into `tags.md` grouped by the Codebook."
    *   *Prompt B:* "Re-read the raw transcripts and generate a fresh `tags.md` based on the Codebook."
3.  **Tag Interactions:** 
    *   Tags appear as colored dots on transcript Quote Cards.
    *   Hovering expands the dots into labeled bubbles.
    *   Double-clicking a Quote Card opens a modal to Add/Edit Tags (up to 3), adjust duration, or "Hide" from transcript view. *(Note: Hiding does not delete from the actual .md file).*

### Phase 5: MDX Generation & Export (The Slicer)
1.  **Generate MDX:** User clicks "Next". UI provides a prompt for Cursor to convert `findings.md` into `report.mdx` (optimizing the code for React rendering, setting clips to load only on 'play' to save memory).
2.  **Download HTML (The Local FFMPEG Slicer):**
    *   User clicks "Download HTML".
    *   A local Next.js API route (`/api/slice`) reads the `start` and `duration` of every quote card.
    *   It silently runs local `ffmpeg` to physically slice the `.mp4` into tiny clip files.
    *   It generates a static `.html` file embedding these physical slices inline for ultimate portability.

### Phase 6: Publishing & The Repo Viewer
1.  **Publish to Notion:** UI generates a prompt for Cursor. Cursor runs a local script to push the final report text and metadata to a shared Notion database (replaces Confluence).
2.  **Archive to Google Drive:** UI generates a prompt. Cursor runs a script to upload all assets (HTML, MDX, sliced clips, JSON) to a designated Google Drive folder (replaces SharePoint). Updates `report.json` with the Drive URLs.
3.  **Report Repo Viewer:** A standalone dashboard (can be hosted on GitHub Pages or Vercel).
    *   Reads the master index of projects.
    *   Shows Project Cards with Metadata (Persona, Date, Product).
    *   Provides Links: [View Findings HTML], [View Tag Board HTML], [View Notion Page].
    *   Left pane contains Search (by title) and Filters (Product, Persona, Researcher).

## 4. UI/UX Refinements (Suggestions for the Open-Source build)

1.  **Auto-Sync (Upgrade from "Refresh" button):** 
    Instead of making the user click "Refresh" when Cursor finishes, you can use Next.js server-side file watching (`chokidar` or React `useSWR` polling). When Cursor saves `findings.md`, the UI updates instantly in real-time. It feels like magic.
2.  **The Waffle Menu:**
    Keep the 9-dot waffle menu in the top left of the Report Builder to allow quick navigation to:
    *   Report Repo Viewer (Home)
    *   Global Codebook Editor
    *   Notion Workspace
    *   Google Drive Archive Folder
3.  **Local FFMPEG Implementation:**
    To keep this easy for open-source users to install via `npm install`, utilize `@ffmpeg/ffmpeg` (FFmpeg WebAssembly). This allows the Next.js frontend to slice the videos directly inside the user's browser without requiring them to install FFMPEG software on their actual computer!

## 5. Summary of Why This is Better
By moving to this architecture on your personal PC, you have created a tool that:
1.  Is completely legal and free of corporate IP.
2.  Costs $0 to operate (No Dovetail subscription, no OpenAI API costs).
3.  Leverages the massive power of Cursor's multi-file context.
4.  Produces physical HTML files with embedded video, which is the holy grail for sharing research with stakeholders who refuse to log into specialized research tools.
