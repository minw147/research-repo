# Codebook Management — Design

**Date:** 2026-03-11
**Status:** Approved

---

## Overview

Three related improvements to the codebook system:

1. **Global codebook editor** — edit shared tags from the UI (not just the filesystem)
2. **CSV import on project creation** — bootstrap a project with a custom codebook
3. **SKILL.md improvements** — split AI prompts for findings.md vs tags.md; add codebook file references

---

## Background: Current Codebook Flow

| Layer | File | Editable today |
|---|---|---|
| Global codebook | `data/global-codebook.json` | Filesystem only |
| Project codebook | `content/projects/[slug]/codebook.json` | CodebookEditor modal |

At runtime, `mergeCodebooks()` combines both. Global tags show a "Global" badge in the UI and are currently read-only. Project tags are fully editable.

## Feature 1: Global Codebook Editor

### Access

- **Header nav** (WorkspaceNav) gets a "Codebook" link with a tag icon, placed next to Help
- The existing **"Codebook" button on the Tags page markdown editor banner is removed**
- Clicking the header link opens the CodebookEditor modal in a root-level portal

### Modal structure

Two tabs: `[ Project Tags ]  [ Global Tags ]`

**Context-awareness:**
- On a project page → opens with Project Tags tab active
- On the dashboard or any non-project page → opens with Global Tags tab active; Project tab hidden

**Project Tags tab** — unchanged from today. Scoped to `content/projects/[slug]/codebook.json`.

**Global Tags tab** — same UI as Project tab, but all tags are editable. Delete and rename trigger a cascade flow (see below).

### Cascade on rename / delete

Before any destructive write to the global codebook, the UI runs a dry-run:

1. `POST /api/codebook/cascade` with `{ dryRun: true, action: "rename" | "delete", oldId, newId? }`
2. API scans all `content/projects/*/tags.md` and `content/projects/*/findings.md`
3. Returns `{ affectedFiles: string[], affectedQuoteCount: number }`
4. UI shows a confirmation modal:
   > "Renaming 'usability' → 'ux-issue' will update 34 quotes across 3 projects. Published reports will need to be re-exported to reflect this change. Continue?"
5. User confirms → real write executes: markdown files patched, then `data/global-codebook.json` saved

**For rename:** old tag ID replaced with new slug in all `tags:` lines
**For delete:** tag ID removed from all `tags:` lines
**Writes:** atomic per-file (read → patch → write)

**Published reports note:** Static HTML exports bake tag labels at export time. Cascade only updates local markdown source files. Already-published reports in SharePoint / Google Drive / local folder become stale and must be re-exported.

### Save behavior

| Tab | Saves to | API |
|---|---|---|
| Project Tags | `content/projects/[slug]/codebook.json` | Existing file API |
| Global Tags | `data/global-codebook.json` | `PUT /api/codebook/global` (new) |

## Feature 2: CSV Import on New Project Creation

### New Project modal — multi-step

**Step 1 — Project details** (unchanged today)

Codebook choice:
- `Default (use global codebook)` — project starts with no project-level overrides
- `Custom (upload CSV)` — proceeds to Step 2

**Step 2 — CSV Upload & Preview** (new, only shown if Custom chosen)

#### CSV format

```
label,category
UI Friction,Pain Point
Mental Model Gap,Cognitive
```

- Header row required
- No `id` column — auto-generated by slugifying label (e.g. "UI Friction" → `ui-friction`)
- No `color` column — auto-assigned round-robin from a fixed 12-color palette, skipping colors already in use
- ID collision: append `-2`, `-3`, etc.

#### Preview table

| Label | Category | ID (auto) | Color | Status |
|---|---|---|---|---|
| UI Friction | Pain Point | ui-friction | 🟠 | ✅ |
| Mental Model Gap | | mental-model-gap | 🔵 | ⚠️ Missing category |

- **Category cell** is a combobox: select from existing categories or type a new name and press Enter to create it
- New categories are shown with a subtle "New" badge
- A summary below the table lists: "New categories to be created: Pain Point, Cognitive"
- Rows with an empty category block the "Create Project" button until resolved

#### Footer note

> "You can update your codebook anytime from the Codebook link in the header."

#### On confirm

Project is created and `content/projects/[slug]/codebook.json` is written with the parsed tags. Global tags remain separate and are merged at runtime as usual.

## Feature 3: SKILL.md & Prompt Improvements

### Problem

Both `findings.md` and `tags.md` prompts currently reference "Steps 1–5" in SKILL.md identically. Step 4 is generic, providing no distinct guidance for the two different output formats. Additionally, SKILL.md has no pointer to codebook file locations, making it hard to use outside the app.

### A — Split Step 4 into two branches

**Step 4a — findings.md (thematic synthesis)**
Write `findings.md`. Create H2 sections per theme. Under each theme, write supporting quotes using the standard quote format. Synthesize patterns, not just individual quotes.

**Step 4b — tags.md (evidence per tag)**
Write `tags.md`. For each tag in the codebook, create an H2 section. Under each tag, collect 3–5 high-quality quote examples that represent that concept. Use the standard quote format.

### B — Add codebook reference to SKILL.md

Add a section near the top of SKILL.md:

```
## Codebook locations

- Global tags: `data/global-codebook.json`
- Project tags: `content/projects/[slug]/codebook.json`

When a prompt does not inline the codebook, read the relevant files directly to discover available tag IDs, labels, and categories before tagging.
```

### C — Update prompt references

| File | Current reference | Updated reference |
|---|---|---|
| findings.md prompts | "Follow Steps 1–5" | "Follow Steps 1–3, then Step 4a" |
| tags.md prompts | "Follow Steps 1–5" | "Follow Steps 1–3, then Step 4b" |

Changes are in `src/lib/prompts.ts` — specifically `buildAnalyzeTranscriptsPrompt`, `buildAnalyzeFindingsPrompt`, and `buildGenerateTagsPrompt`.

## ID & Color Generation

**ID slugification:**
- `"UI Friction"` → `"ui-friction"` (lowercase, spaces to hyphens, strip special chars)
- Collision check against global + project tags
- If collision: append `-2`, `-3`, etc.

**Color palette:**
- 12 distinct colors defined in `src/lib/color-themes.ts`
- Assigned round-robin per new tag
- Skips colors already in use within the same codebook

## Out of Scope

- Editing the global codebook from outside the app (filesystem edits remain valid)
- Re-publishing stale reports automatically after a cascade (user must re-export manually)
- Importing JSON codebooks (CSV only)
- Per-project override of global tag colors
